# Spec-First Protocol Examples

This directory contains examples of locked specifications generated by the
Spec-First Protocol. They demonstrate how SFP captures requirements,
handles constraints, and structures outputs across different domains.

## Software Engineering

- **[Skill Distribution Utility](2026-06-01_SKILL-DISTRIBUTION_SPEC.md)**
  A specification for the lightweight CLI tools and scripts used to install
  SFP's agent skills across different IDEs and agent frameworks. Demonstrates
  how SFP handles path resolution, OS-specific edge cases, and failure modes.

- **[SFP Documentation Website](2026-06-01_SFP-UI_SPEC.md)**
  A specification for the static HTML/CSS/JS documentation site for the
  Spec-First Protocol (the website you are likely reading this on). Demonstrates
  front-end UI requirements, styling tokens, and layout guidelines.

## Domain-Agnostic (Non-Software)

- **[Disney Vacation Planning](non-software/2026-06-02_PERFECT-DISNEY-VACATION_SPEC.md)**
  A comprehensive plan for a 4-day family vacation to Walt Disney World. This
  example proves that SFP is fundamentally domain-agnostic. It extracts budget
  constraints, pacing preferences, and logistical requirements, producing not
  just a schedule but also:
  - [Booking Checklist](non-software/Disney_Booking_Checklist.md)
  - [Packing List](non-software/Disney_Packing_List_Family4.md)
  - [Discovery Notes](non-software/discovery_notes.md)
